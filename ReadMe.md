# Feed GG

League of Legends のプレイヤー情報を検索して、プロフィール、ランク、対戦履歴を表示するアプリ。

## 概要

ユーザーは LoL のリージョン、Riot ID の `gameName`、`tagLine` を入力してプレイヤーを検索できる。
検索時に Riot API から必要な情報を取得し、取得結果を DB に保存する。
一度保存したプレイヤーは再検索時に DB から表示できるようにし、プロフィール画面の更新ボタンから最新状態を再取得できるようにする。

## MVP で実現したいこと

1. プレイヤーを Riot ID で検索できる
2. プレイヤーの基本プロフィールを表示できる
3. プレイヤーの現在ランクを表示できる
4. プレイヤーの直近対戦履歴を表示できる
5. 取得したデータを DB に保存できる
6. 保存済みプレイヤーを再検索時にすぐ表示できる
7. 更新ボタンで Riot API から最新情報を再同期できる

## 検索条件

入力項目:

- リージョン
  - 例: `JP1`
- `gameName`
  - Riot ID の名前部分
- `tagLine`
  - Riot ID のタグ部分

検索対象:

- Riot ID で一意にプレイヤーを特定する
- 内部的には Riot API から `puuid` を取得して以降のデータ参照に使う

## 画面要件

### 1. 検索画面

表示内容:

- リージョン入力
- `gameName` 入力
- `tagLine` 入力
- 検索ボタン

動作:

1. ユーザーが検索条件を入力して検索する
2. そのプレイヤーが DB に存在する場合は、保存済みデータを使ってプロフィール画面を表示する
3. DB に存在しない場合は Riot API からデータを取得し、保存後にプロフィール画面を表示する

### 2. プレイヤープロフィール画面

表示したい情報:

- プロフィールアイコン
- Riot ID
  - `gameName#tagLine`
- リージョン
- `puuid`
  - 内部では必須
  - UI に出すかどうかは任意
- サモナーレベル
- 最終同期日時
- 更新ボタン

動作:

- 更新ボタンを押したら Riot API を再実行して最新のプロフィール、ランク、対戦履歴を取り直す
- 再取得した内容は DB を更新する

### 3. ランク表示

表示したい情報:

- キュー種別
  - 例: Solo/Duo, Flex
- tier
  - 例: GOLD, PLATINUM
- rank
  - 例: I, II, III, IV
- LP
- 勝利数
- 敗北数
- 必要なら昇格戦情報

想定:

- 1 プレイヤーに対して複数ランクキューが返るので、キューごとに表示する
- MVP では `RANKED_SOLO_5x5` と `RANKED_FLEX_SR` を優先表示できればよい

### 4. 対戦履歴表示

表示したい情報:

- 試合 ID
- プレイ日時
- ゲームバージョン
- キュー種別
- 使用チャンピオン
- レーンまたはロール
- 勝敗
- KDA
- サモナースペル
- 試合時間

想定:

- MVP では直近 20 件程度を表示対象にする
- 各試合について、対象プレイヤー本人の参加情報を優先表示する

## データ取得フロー

検索時の基本フロー:

1. Riot ID から `puuid` を取得する
2. `puuid` を使ってプレイヤーのプロフィール情報を取得する
3. `puuid` を使ってランク情報を取得する
4. `puuid` を使って対戦履歴の match id 一覧を取得する
5. 各 match id の試合詳細を取得する
6. アイコン画像 URL を組み立てる
7. 取得結果を DB に保存する

更新時の基本フロー:

1. プロフィール画面で更新ボタンを押す
2. Riot API を再実行する
3. DB のプレイヤー情報、ランク情報、対戦履歴を更新する

API の詳細は [docs/riot-api-spec.md](./docs/riot-api-spec.md) にまとめる。

## DB 保存方針

最低限保存したい単位:

1. プレイヤー
2. ランクエントリ
3. 試合
4. 試合参加者

一意キーの考え方:

- プレイヤー: `puuid`
- ランク: `puuid + queueType`
- 試合: `matchId`
- 試合参加者: `matchId + puuid`

保存したい理由:

- 再検索時に API を毎回叩かずに表示できる
- 直近の取得結果をキャッシュとして使える
- 更新ボタンで差分更新しやすい

## 検索時の挙動

### DB に存在しないプレイヤー

- Riot API を呼ぶ
- 取得した情報を DB に保存する
- 保存後にプロフィール画面を表示する

### DB に存在するプレイヤー

- DB の情報をそのまま使ってプロフィール画面を表示する
- MVP では再検索だけでは自動再取得しない
- 最新化は更新ボタンで行う

## 更新ボタンの仕様

更新ボタン押下時:

1. 対象プレイヤーの `puuid` を使って Riot API を再取得する
2. プレイヤー情報を上書き更新する
3. ランク情報を上書き更新する
4. 対戦履歴を追加または更新する
5. 更新後の内容を画面に再表示する

## エラーハンドリング

最低限考慮したいケース:

- 入力された Riot ID が存在しない
- リージョンが不正
- Riot API の rate limit に達した
- Riot API が一時的に失敗した
- DB 保存に失敗した

MVP の挙動:

- ユーザーにエラーメッセージを表示する
- 保存途中で失敗した場合は中途半端な状態を避けたい
- 更新時に失敗した場合は既存 DB データを壊さない

## MVP では後回しにするもの

- ユーザー認証
- お気に入り登録
- 複数プレイヤー比較
- 試合詳細の高度な分析
- ライブゲーム情報
- バックグラウンド自動更新

## 実装メモ

- プレイヤー指定は Riot ID 入力に統一する
- 内部キーは `puuid` ベースで扱う
- ランクは queue ごとに別レコードとして持つ
- 対戦履歴は一覧表示しやすいように、試合本体と参加者情報を分けて保存する
- プロフィールアイコンは Riot API 本体ではなく Data Dragon を使う
